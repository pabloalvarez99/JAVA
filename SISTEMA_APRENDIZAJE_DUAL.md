# SISTEMA DE APRENDIZAJE DUAL - PREPARACION PARA EXAMEN POO

---

## QUE ES ESTE SISTEMA

Este repositorio implementa un **sistema pedagogico de dos fases** disenado especificamente para prepararte para el examen de Programacion Orientada a Objetos.

**La filosofia:**
1. **FASE 1:** Aprendes CON tutoriales completos (Ejercicios 01-10)
2. **FASE 2:** Practicas SIN soluciones (Ejercicios 11-20)

**Por que funciona:**
- La Fase 1 te da las herramientas y patrones
- La Fase 2 te obliga a aplicarlos independientemente
- Esta progresion simula como sera el examen: tendras que resolver problemas NUEVOS usando conceptos CONOCIDOS

### Como estudiar la Fase 1:

**Primera pasada (Dias 1-3):**
1. Lee cada ejercicio completo ANTES de programar
2. Entiende el contexto y requisitos
3. Estudia el PASO A PASO
4. Escribe el codigo siguiendo el tutorial
5. Compila y prueba
6. Compara tu codigo con la solucion

**Segunda pasada (Dias 4-6):**
1. Intenta resolver SIN mirar la solucion
2. Usa el tutorial solo si te atascas
3. Compara tu implementacion con la oficial
4. Identifica que partes te costaron

**Tercera pasada (Dias 7-9):**
1. Resuelve de memoria
2. Cronometra tu tiempo
3. Objetivo: terminar en el tiempo estimado
4. Revisa la seccion "Errores Comunes"

**INDICADOR DE DOMINIO:**
Puedes resolver cada ejercicio en el tiempo estimado SIN mirar el tutorial.

```
└── Ejercicio20_SimuladorExamen.md
```

### Que incluyen:

**NINGUN ejercicio de la Fase 2 tiene soluciones. Solo:**
1. Contexto del problema
2. Requisitos funcionales
3. Requisitos tecnicos (que patrones usar)
4. Formato de archivos de entrada
5. Formato de salida esperada
6. Datos de prueba

**NO incluyen:**
- Paso a paso
- Codigo de solucion
- Explicaciones de implementacion

### Por que NO hay soluciones:

**Razon pedagogica:**
El examen te dara un problema NUEVO. No podras consultar soluciones. Por lo tanto, debes practicar resolviendo problemas sin ayuda.

**Si te atascas:**
1. Revisa los ejercicios de la Fase 1 para ver patrones similares
2. Identifica CUAL patron se aplica
3. Adapta la solucion de la Fase 1 al nuevo contexto
4. NO copies y pegues - entiende y reescribe

**Metodo de trabajo:**
```
1. Leer problema completo
2. Identificar patrones necesarios
3. Disenar arquitectura en papel
4. Buscar ejercicio SIMILAR en Fase 1
5. Adaptar la solucion (NO copiar directamente)
6. Implementar
7. Probar con datos proporcionados
8. Ajustar hasta que funcione
```

### Progresion de dificultad:

**Ejercicios 11-13 (Basicos):**
- Equivalente a Ejercicios 01-04
- Un patron principal
- Archivo simple
- Tiempo: 30-45 minutos

**Ejercicios 14-16 (Intermedios):**
- Equivalente a Ejercicios 05-07
- Dos patrones combinados
- Archivos mas complejos
- Tiempo: 60-90 minutos

**Ejercicios 17-20 (Avanzados):**
- Equivalente a Ejercicios 08-10
- Tres o mas patrones
- Archivos complejos
- Calculos estadisticos
- Tiempo: 2-3 horas

**Ejercicio 20 es un SIMULACRO COMPLETO del examen.**

### Como estudiar la Fase 2:

**IMPORTANTE:** Solo empieza la Fase 2 cuando hayas completado TODA la Fase 1.

**Estrategia:**
1. Elige un ejercicio de tu nivel actual
2. Lee el enunciado COMPLETO dos veces
3. Identifica los patrones necesarios
4. Diseña la arquitectura en papel (15-20 min)
5. Implementa sin mirar soluciones
6. Prueba con los datos proporcionados
7. Si la salida no coincide, depura tu codigo
8. Si te atascas >30 min en un punto, revisa Fase 1

**Criterio de exito:**
Tu salida debe coincidir EXACTAMENTE con la esperada.

**Que hacer si no sale:**
1. Revisa el formato de salida (espacios, precision de decimales)
2. Verifica el parsing del archivo
3. Comprueba las formulas de calculo
4. Imprime valores intermedios para debug
5. Compara tu arquitectura con Fase 1

## ESTRATEGIA PARA EL DIA DEL EXAMEN

### Antes del examen:
1. Dormir bien (minimo 7 horas)
2. Comer algo ligero
3. Llevar botella de agua
4. Llegar 15 minutos antes
5. NO revisar codigo en los ultimos 30 minutos (te confundira)

### Durante el examen:

**Primeros 5 minutos:**
1. Respirar profundo
2. Leer TODO el enunciado
3. Leer de nuevo, mas despacio
4. Identificar: problema similar a cual ejercicio?

**Minutos 5-20 (PLANIFICACION):**
1. Dibujar jerarquia de clases
2. Listar patrones requeridos
3. Disenar arquitectura del Sistema
4. Escribir estructura de clases (solo nombres y metodos principales)
5. Planificar orden de implementacion

**Minutos 20-150 (IMPLEMENTACION):**
1. Empezar por la jerarquia base
2. Implementar patrones uno a uno
3. Compilar cada 15-20 minutos
4. Probar con datos simples primero
5. NO te obsesiones con un bug >15 min (sigue adelante)

**Ultimos 30 minutos (VALIDACION):**
1. Probar con TODOS los datos proporcionados
2. Verificar formato de salida EXACTO
3. Agregar comentarios faltantes
4. Revisar que se cumplan TODOS los requisitos
5. Hacer ZIP y subir (dejar 5 min para esto)

**Gestion del tiempo:**
- Problema 1 (Ruteo): 30-45 minutos
- Problema 2 (Programacion): 2.5 horas
- Buffer para imprevistos: 15 minutos

## ERRORES COMUNES DE ESTUDIANTES

### Error 1: No practicar Fase 2
"Ya hice todos los tutoriales, estoy listo"
**Realidad:** Si solo haces la Fase 1, no sabes si REALMENTE puedes resolver problemas solos.

### Error 2: Solo hacer Fase 2
"Voy a aprender haciendo, sin tutoriales"
**Realidad:** Te frustras, programas mal, aprendes anti-patrones.

### Error 3: No cronometrar
"Lo importante es terminarlo, no importa cuanto tarde"
**Realidad:** El examen tiene limite de tiempo. Si tardas 5 horas en Ejercicio 10, reprobaras.

### Error 4: No revisar errores comunes
"Mi codigo funciona, ya esta"
**Realidad:** Puede funcionar por casualidad, o con ciertos datos. Los errores comunes te preparan para casos especiales.

### Error 5: Estudiar la noche antes
"Tengo buena memoria, lo leo todo el dia antes"
**Realidad:** POO se aprende HACIENDO, no leyendo. Necesitas las 3 semanas completas.

### Error 6: No hacer el simulacro
"Ya hice Ejercicio 10, no necesito el 20"
**Realidad:** El simulacro tiene diferencias sutiles que te preparan para variaciones del examen.

## FILOSOFIA PEDAGOGICA

Este sistema se basa en principios probados de aprendizaje:

1. **Scaffolding (Andamiaje):**
   - Fase 1 te da soporte completo
   - Fase 2 retira el soporte gradualmente
   - Como aprender a andar en bicicleta: primero con rueditas, luego sin ellas

2. **Deliberate Practice (Practica Deliberada):**
   - No basta repetir lo mismo
   - Cada ejercicio es ligeramente mas dificil
   - Trabajas justo en el limite de tu capacidad

3. **Spaced Repetition (Repeticion Espaciada):**
   - Ves el mismo patron en diferentes contextos
   - Strategy aparece en Ejercicios 02, 07, 08, 10, 14, 16, 17...
   - Cada vez lo entiendes mas profundo

4. **Retrieval Practice (Practica de Recuperacion):**
   - Fase 2 te obliga a RECORDAR, no solo reconocer
   - El esfuerzo de recordar fortalece la memoria
   - Simula la condicion del examen

5. **Transfer of Learning (Transferencia de Aprendizaje):**
   - Aprendes a aplicar conceptos en nuevos contextos
   - No solo memorizas una solucion
   - Desarrollas habilidad de "pensar como programador"
